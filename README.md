# banking
# 📊 Banking Data Analysis using SQL

## 📌 Project Overview
This project focuses on analyzing a banking dataset using SQL to extract meaningful insights about customers, their financial behavior, and marketing campaign effectiveness.

## 🛠️ Tools Used
- PostgreSQL
- pgAdmin 4
- SQL

## 📂 Dataset Details
The dataset contains customer information such as:
- Age
- Job
- Marital Status
- Education
- Account Balance
- Loan & Housing Status
- Contact Details
- Campaign Data

## 🧱 Database Structure
A table named `test` is created with the following columns:
- age, job, marital, education
- balance, housing, loan
- contact, day, month
- duration, campaign, pdays
- previous, poutcome

## 🔍 Key SQL Operations Performed

### ✅ Basic Queries
- Retrieve all records
- Select specific columns
- Filter high balance customers

### ✅ Filtering
- Customers based on marital status
- Loan and housing conditions

### ✅ Aggregations
- Average balance
- Total customers
- Max & Min age

### ✅ Group By Analysis
- Customers by job role
- Average balance by education

### ✅ Advanced Analysis
- Top 5 customers with highest balance
- Campaign month analysis
- Customers with no previous contact

### ✅ Interview-Level Insights
- Job with highest average balance
- Customer distribution by marital status

## 📈 Key Insights
- Identified high-value customers based on balance
- Analyzed customer demographics
- Evaluated marketing campaign effectiveness

## 🚀 How to Run
1. Create the table using the provided SQL script
2. Import the dataset (CSV file)
3. Run queries from `banking.sql`
